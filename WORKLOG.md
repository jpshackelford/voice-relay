- **Labels added:** `ready`, `priority:high`
- **Root cause (verified against `60068b2`):** Four interacting gaps — (1) no startup pass re-attaches `AgentDriver` from `session.metadata.aiConversationId`; (2) `shouldAutoConnect`'s `isFirstDevice` heuristic in `server/src/auto-connect.ts:197` is false post-restart because `session_devices` rows persist; (3) `OpenSessionOpts` has no `existingConversationId` so `openSession` always creates a new OH conversation (orphaning the original); (4) the WS `text` handler in `server/src/index.ts:736` silently drops messages when `agentDriver.hasSession` is false.

- **Files affected:** `session-repository.ts`, `agent-driver/types.ts`, `agent-driver/openhands.ts`, `openhands.ts`, `auto-connect.ts`, new `agent-rehydrate.ts`, `index.ts`, plus tests.
- **Migration concerns:** None — purely additive; `metadata.aiConversationId` already persisted; `existingConversationId` is optional so existing call sites compile unchanged.
